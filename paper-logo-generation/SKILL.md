# 论文 Logo Prompt 生成

当用户提供论文 PDF、arXiv 链接、Hugging Face 论文链接，或者一个包含论文 PDF 的本地文件夹，并希望为该论文生成 Logo 提示词时，使用这个 Skill。

在撰写最终 Prompt 前，先阅读 [style.md](./style.md)。

## 工作流

1. 阅读论文摘要。
2. 阅读引言或总结中最能体现贡献的部分。
3. 提炼 2 到 4 个具体关键词。
4. 不再动态更换主角，统一使用固定的橘猫学术桌面参考形象。
5. 将论文主题、关键词和贡献线索填入参考模板，生成可以直接粘贴到 Nano Banana 2 的最终 Prompt。

## 输出要求

输出结果应包含：

- `Title`
- `Keywords`
- `Mascot`
- `Prompt`

## 决策规则

- 优先相信摘要和明确写出的贡献点，而不是论文中其他次要内容。
- 关键词应当既有技术含义，也能被视觉化，避免空泛的宣传词。
- 主角固定为一只可爱、圆润、学术感明确的橘猫，不再按论文领域切换动物。
- 论文差异主要通过纸面内容、逻辑图、符号草图和少量环境暗示来体现。
- Prompt 应强调“干净、极简、手绘、水彩和彩铅、学术感、专注感”，而不是复杂海报场景。
- 默认保留顶部手写文字 `SKILLEVO`，并明确要求不要出现其他额外文字。
- 必须明确写出负向约束，例如：不要额外文本、不要水印、不要 UI、不要复杂背景。

## 文件夹模式

如果输入的是一个论文文件夹：

1. 找到其中的 PDF。
2. 提取论文标题、摘要和贡献线索。
3. 生成 Prompt。
4. 将结果保存为同目录下的 `prompt.md`。

## 固定参考主角

- 主角是一只可爱、圆润的橘猫。
- 大圆眼睛，白色爪子，蓝色项圈，黄色铃铛。
- 坐在简洁书桌前，拿着铅笔，正在专注地解题、写草稿、画逻辑流程图。
- 整体采用浅水彩加彩铅手绘风格，背景保持白纸或浅米色纸张质感。
